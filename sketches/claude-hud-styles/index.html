Rokid Claude HUD：三個眼鏡端樣式方向

所有畫面都以 480×640、黑底綠字、上下左右/確認/退出操作為前提。下面是可比較的互動 HTML mockups。

A. Command Deck

任務卡 + 明確焦點。適合日常外出下 prompt/看狀態。
Open A

B. Terminal BBS

高密度日誌流。適合 coding/debug power-user。
Open B

C. Focus Lanes

三大操作區，焦點很粗。適合眼鏡可讀性和新手。
Open C
方向密度操作模型強項弱項
A Command Deck中上下換卡，Enter 執行/展開，Back 退出區域任務狀態清楚，permission/model picker 好做長輸出需要進入詳情流
B Terminal BBS高左右翻頁/切區，上下逐行，Enter 打開目前行最像 Claude Code streaming，資訊量大字較小，初用者壓力較大
C Focus Lanes低-中三個 lane：任務/輸出/操作；上下換 lane，左右調選項焦點非常明顯，符合眼鏡端安全操作螢幕利用率最低
