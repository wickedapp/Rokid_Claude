# Rokid Claude HUD style sketches

Disposable design sketches for a 480×640 Rokid glasses HUD. All variants are intentionally constrained around glasses controls: **Enter/confirm**, **Back/exit**, and **Left/Right/Up/Down style focus/scroll**.

Open the comparison page:

```bash
open /Volumes/DATA/Development/Rokid_Claude/sketches/claude-hud-styles/index.html
```

Variants:

1. `variant-a-command-deck.html` — task-card command deck, action/focus first.
2. `variant-b-terminal-bbs.html` — dense terminal/BBS stream, maximum information.
3. `variant-c-focus-lanes.html` — large focus lanes for safe directional navigation.

Design assumption: real Rokid input is limited; do not rely on touch targets, tiny tabs, long-press, hover, or text fields. The UI should always expose the current focused region and what Enter/Back/Left/Right do.
