## Variant: Terminal BBS

### Design stance
Power-user terminal: maximize live event density while preserving a visible row focus.

### Key choices
- Layout: top tabs + BBS table stream + bottom key map.
- Typography: compact monospace; one event per row.
- Interaction: Left/Right switches tabs; Up/Down moves event focus; Enter opens row details; Back closes details.

### Trade-offs
- Strong at: debugging, coding tasks, seeing many tool events at once.
- Weak at: eye fatigue; smaller text is less forgiving on optical displays.

### Best for
When this becomes a real Claude Code HUD for technical users who want to inspect tool calls and logs.
